Contributing
============

We welcome issues, feature requests, and pull requests to improve **sqCAD**.

Getting started
---------------

- Check the `README` on GitHub for the latest installation and project overview.
- Install the package in editable mode with the docs and dev extras (once available).
- Run the test suite before submitting changes.

How to report a bug
-------------------

When opening a GitHub issue, please include:

- Your sqCAD version and Python version.
- A minimal code snippet or notebook that reproduces the problem.
- Any error messages and a short description of the expected behavior.

Submitting a pull request
-------------------------

- Fork the repository and create a feature branch from ``main``.
- Keep changes focused and add or update tests when possible.
- Update the documentation (Quickstart, API, or other pages) if your change affects user-facing behavior.
- Open a pull request describing the motivation and the main changes.

Code style
----------

- Follow the existing style of the surrounding code.
- Prefer small, well-documented functions and classes with clear docstrings.
- Type hints are encouraged where they help clarify interfaces.
